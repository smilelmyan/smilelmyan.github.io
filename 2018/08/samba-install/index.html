samba文件共享

  2018-08-02   i3桌面环境  Comments Word Count: 448words Read Count: 1minutes

搭建文件分享服务，方便和其它设备共享文件。

安装

$ sudo pacman -S samba

配置

获取默认配置文件

$ sudo wget "https://git.samba.org/samba.git/?p=samba.git;a=blob_plain;f=examples/smb.conf.default;hb=HEAD" -O /etc/samba/smb.conf

编辑 /etc/samba/smb.conf，折腾了好几个小时，才发现自己需要的功能是如此好配置。

大部分参数默认就不贴了，注释掉不用的打印功能。就改动两处

修改一行

log file =  /var/log/samba/log.%m

在[homes]中添加一行

valid users = myname

添加myname的samba密码，myname是我的linux账户名

$ sudo smbpasswd -a myname

启动smb服务

$ sudo systemctl enable smb.service
$ sudo systemctl start smb.service

用虚拟机安装的windows使用NAT连接可直映射

\\host ip\homes

输入刚才的账户密码。至此home目录下的内容就可以被windows访问了。

说明

模板中[homes]默认开启用户家目录的共享，需要在原配置中增加

   valid users = %S

%S表示所有用户，如果只有我一个人，改成自己的用户名就好了。

其它共享文件的方案需注意，path后的路径，每一层权限都要设置为777，要不进不去。 对于这种public模式的共享，须在[global]中添加

security = user
map to guest = Bad User

4.xx的版本设置方法，可实现匿名登录，默认匿名用户名是nobody。
